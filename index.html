Architecture

Subsystems

Every subsystem lives under clz::, follows init() → update() → shutdown(), and stays decoupled from its neighbours by design.

Renderer
Vulkan 1.3 pipeline. Dynamic rendering, per-frame command buffers, swapchain management, and sync primitives.
@curl0z
Window
GLFW lifecycle, keyboard input, and Vulkan surface creation. Forwards to native and render layers.
@curl0z
ECS
Custom entity-component system. Typed component storage, scene management, entity lifecycle. Zero external deps.
@curl0z
Math
Header-only vec2, vec3, vec4, mat4x4. Written to match the engine's type conventions exactly.
@curl0z
Scripting
Lua bridge for game logic. Run scripts without recompiling. Exposes engine state and subsystem calls to Lua.
@curl0z
Config
TOML and JSON loading. Drive engine settings from config/engine.toml without a recompile.
@curl0z
Physics
Jolt Physics integration for rigid body simulation and collision. Foundation planned, ready to be built out.
planned
Audio
OpenAL Soft for 3D positional audio. Subsystem scaffold is in place, implementation open for a contributor.
unassigned

BASH

sudo apt install cmake ninja-build clang-format libvulkan-dev vulkan-tools

# Clone with submodules
git clone --recursive https://github.com/curl0z/curloz-engine.git
cd curloz-engine

# Configure and build
cmake -B build/debug -G Ninja
cmake --build build/debug

Maintainer Guide

Contributing to Curloz Engine

Read this before writing a single line. The engine is built as a collection of independent subsystems under clz::. Every subsystem owns its own state. They don't reach into each other; they communicate through defined interfaces only.

!
Boundary rule: Do not touch code outside your assigned area without discussing it first. If you need something from another subsystem, ask. Don't patch it yourself.
Design

Architecture

Every subsystem that requires it exposes three functions: init(), update() called once per frame, and shutdown(). Shutdown always runs in reverse init order. Not every subsystem needs all three.

Decoupling is intentional. The renderer doesn't know about ECS internals, it receives only the component data it needs. main.cpp is the only file that sees all subsystems simultaneously, calling each in dependency order at startup, every frame, and in reverse at the end.

PROJECT LAYOUT
curloz-engine/
├── src/renderer/       ← Renderer subsystem
├── src/window/         ← Window subsystem
├── src/ecs/            ← Entity Component System
├── src/audio/          ← Audio subsystem
├── src/scripting/      ← Lua scripting bridge
├── include/math/       ← Header-only math library
├── include/core/       ← Engine-wide utilities
└── src/main.cpp        ← Orchestrates all subsystems

Workflow

Git Branches

Never commit directly to main. Always branch, then PR.

mainAlways stable, always builds. Protected.
feature/xxxNew features. Branch from main, PR back to main.
fix/xxxBug fixes. Same pattern as feature branches.

Windows

BRANCH LIFECYCLE
# Start work
git checkout -b feature/physics-rigidbody

# Push branch
git push --set-upstream origin feature/physics-rigidbody

# After merge, delete
git branch -d feature/physics-rigidbody
git push origin --delete feature/physics-rigidbody
Workflow

Commit Messages

Format: type: short description. Types are feature, fix, refactor. No "fixed stuff". No "wip" on main. No giant commits that touch everything.

EXAMPLES
feature: add rigid body component to ECS
fix: resolve swapchain recreation on window resize
refactor: clean up command buffer recording in mainloop
Workflow

Pull Requests

Every branch merge, even between collaborators, goes through a PR. Write what you did and why in the description. Minimum one review before merge.

i
If your PR touches more than one subsystem, tag the respective owners in the description. They need to review their section before merge.

Code Style

Enforced by .clang-format and .editorconfig in the repo root. Set up the pre-commit hook so formatting runs automatically on every commit.

Indentation
Tabs, 8-wide
Braces
Allman, brace always on its own line
Pointers
Left-aligned: int* ptr
Functions
camelCase
Structs / Classes
PascalCase
Members / Variables
camelCase
Subsystem Variable Prefix
Every module-level variable is prefixed by subsystem e.g. ph_body for Physics, rn_device for Renderer, wn_window for Window

Linux: hook setup

BASH
# Install clang-format
sudo apt install clang-format    # Ubuntu
sudo emerge -av clang            # Gentoo

# Pre-commit hook
cp scripts/pre-commit .git/hooks/pre-commit
chmod +x .git/hooks/pre-commit

# Fix formatting manually
find src include -name "*.cpp" -o -name "*.hpp" | xargs clang-format -i

Windows: Visual Studio

Enable via Tools → Options → Text Editor → C/C++ → Code Style → Formatting → Enable ClangFormat support → Format on Save. VS picks up .clang-format from the repo root automatically.

        glm::quat rotation; ///< Rotation as quaternion
        glm::vec3 scale;    ///< Non-uniform scale
